# HUD TAG GLOSSARY — KEEPING TIME
## The Score: Authoritative Tag Reference

> **Version**: 1.0 — Compiled from confirmed Volume One manuscript tags + Writer's Guide v2 rules
> **Purpose**: Single reference for all HUD tag formats, categories, confirmed exemplars, and prohibited terms
> **Use with**: `KEEPING_TIME_Writers_Guide_v2.md` (§1), `KEEPING_TIME_Analysis_Prompt_v2.md` (Appendix B)

---

## HOW TO USE THIS GLOSSARY

**When writing**: Check the relevant category for approved tag structures before writing a new HUD line.
**When editing**: Use the Prohibited Tags section to scan for violations, then use the replacement table.
**When auditing**: Each template Section B pass should cross-reference the tags found in a chapter against this glossary.

Tags are organized by functional category. Each entry shows:
- `FORMAT` — the approved structure
- `EXEMPLAR` — a confirmed Volume One usage (if available)
- `NOTES` — writing guidance

## PART 1: APPROVED TAG CATEGORIES

### CAT-01: BIOMETRIC / PHYSIOLOGICAL

Tags that report Taro's physical state as data. These replace all soft emotional language.

**Status**
```
> STATUS: VALUE [Modifier]
> STATUS: STUNNED [CAUSE: FORCED_RESET]          ← Ch. 7 exemplar
> STATUS: THERMAL_LOAD [REGISTER: HIGH // RATE: CRITICAL]    ← replaces SENSATION: BURNING_WIRE
> STATUS: NERVE_DAMAGE [CRITICAL]                ← replaces SENSATION: PAIN
> STATUS: LIMITER_REMOVED [AMPLITUDE: UNCAPPED]  ← replaces SENSATION: FREE_FALL
```

**Biometric**
```
> BIOMETRIC: VALUE [Modifier]
> BIOMETRIC: VAGAL_RESPONSE [SIGNAL: CORRUPTED]  ← replaces SENSATION: NAUSEA
> BIOMETRIC: CORTISOL_SPIKE [+400%]
> BIOMETRIC: CARDIAC_RATE [BPM: 160]
```

**Output / capacity**
```
> OUTPUT: PERCENTAGE [Modifier]
> OUTPUT: 80%                     ← Ch. 7 exemplar
> OUTPUT: 95% [CRITICAL]          ← Ch. 5 exemplar (pre-correction)
> OUTPUT: 20%                     ← Ch. 5 exemplar
> THERMAL_LOAD: PERCENTAGE        ← Ch. 6 exemplar: > THERMAL_LOAD: 98% [CRITICAL]
> CAPACITY: STATUS                ← Ch. 5: > NODE_STATUS: VEN [CHARGED] / > CAPACITY: 200%
> CPU_DRAIN: PERCENTAGE
> RESERVOIR: STATUS               ← for Damping bands: > RESERVOIR: DEPLETING [RATE: 2%/HR]
```

**NOTES**: All physical states are reported numerically or as system statuses. "Overwhelming," "exhausted," "in pain" never appear as narrative description when the HUD is active — they are data.

### CAT-02: EMOTIONAL / COGNITIVE STATE

Tags that report internal state as system diagnostics. These do not describe feeling; they report system condition.

**Internal conflict**
```
> LOGIC_ERROR // CONTRADICTION_DETECTED          ← for confusion
> LOGIC_FAILURE / EMOTIONAL_OVERRIDE             ← Ch. 1 exemplar — confirmed exemplary
> LOGIC_ERROR: RECURSION [DEPTH: CRITICAL]       ← for spiraling thought
> ANALYSIS: INCONCLUSIVE [DATA: INSUFFICIENT]    ← for genuine uncertainty
```

**Emotional spike (suppressed)**
```
> EMOTIONAL_SPIKE: SUPPRESSED [COLLAR: ACTIVE]   ← recommended addition for Ch. 2
> EMOTIONAL_SPIKE: SUPPRESSED [DAMPING: ACTIVE]
> EMOTIONAL_SPIKE: UNSUPPRESSED [OVERRIDE: TRUE] ← for Resonance Events
```

**Processing state**
```
> PROCESSING: VALUE
> PROCESS: NAI_BOOT [STAGE: 1/3]    ← replaces SENSATION: BOOT_SEQUENCE
> PROCESS: QUERY_RUNNING            ← Taro actively searching
> DIAGNOSTIC: BASELINE_ALTERED      ← Ch. 7 area, confirmed present
> MOTOR_LOOP: ACTIVE [FUNCTION: CALIBRATION]   ← for stimming
> CPU_DRAIN: PERCENTAGE             ← for post-mask exhaustion
```

**System error (for meltdowns / Resonance Events)**
```
> RESONANCE_EVENT [THRESHOLD: EXCEEDED]
> BUFFER_OVERFLOW [Type: Fatal]
> TOTAL_OUTPUT: VALUE [LETHAL]      ← Ch. 5 exemplar
```

**NOTES**: `> THOUGHT:` and `> FEELING:` are always prohibited. Use error codes and system states to render internal experience.

### CAT-03: SOCIAL / ENVIRONMENTAL SCANNING

Tags generated by The Score's passive and active scanning of Taro's environment.

**Target tagging**
```
> TARGET: NAME [CLASS: TYPE]
> OPPONENT: MARCUS_STACCATO [CLASS: RHYTHM_OPERATOR] [THREAT: SEVERE]  ← Ch. 7 exemplar
> TARGET: VOSS [CLASS: Admin]
> TARGET: UNREGISTERED_RESIDENT [STATUS: STARVING] [HEALTH: CRITICAL]  ← Ch. 8 exemplar
```

**Intent analysis**
```
> INTENT: EMOTION [CONFIDENCE: XX%]
> INTENT: DECEPTION [CONFIDENCE: 94%]
> INTENT: THREAT [CONFIDENCE: 78%]
> ANALYSIS: SELECTIVE_OMISSION_ACTIVE   ← Ch. 8 exemplar — C-Order filter system
> ANALYSIS: NEW_TOPOLOGY [POTENTIAL: SIEGE_BREAKER] [EFFICIENCY: HIGH]  ← Ch. 5 exemplar
```

**Micro-expression**
```
> MICRO-EXPRESSION: EMOTION [Modifier]
> MICRO-EXPRESSION: CONTEMPT [SUPPRESSED]
> MICRO-EXPRESSION: RAGE [SUPPRESSED]
```

**Social / affect tags**
```
> TAG: EMOTION [CONFIDENCE: XX%]
> TAG: DISGUST [8.5%]    ← Ch. 4 exemplar
```

**Environmental alert**
```
> ALERT: TYPE
> ALERT: EXTERNAL_PROBE_DETECTED [SOURCE: H.CHORD]   ← Ch. 4 exemplar
> ALERT: COLLISION_IMMINENT                           ← Ch. 7 exemplar
> ALERT: IMPEDANCE_MISMATCH
```

**Ambient scan**
```
> AMBIENT_SCAN: TYPE
> AMBIENT: MIDDLE_C [EFFECT: ENTRAINMENT]    ← replaces SENSATION: GOLDEN_SYRUP
> AMBIENT: SIGNAL_DENSITY [CHANNELS: 14]
```

**Status / disposition**
```
> STATUS: MARKED [TARDY]          ← Ch. 4 exemplar
> STATUS: MASKED                  ← Ch. 14 exemplar: > SIGNAL: MASKED
> MASKING_PROTOCOL: MAX_LOAD      ← Ch. 8 exemplar
> DAMPING: ACTIVE [THERMAL_LOAD: HIGH]
```

**NOTES**: Every time Taro encounters a new character for the first time, The Score should tag them. This is the social prosthetic function — it must not be omitted.

### CAT-04: PHYSICS / COMBAT

Tags that report the real-time mechanics of a resonance encounter.

**Waveform output**
```
> TOPOLOGY: VALUE                  ← describes the waveform shape
> TOPOLOGY: SAWTOOTH_BURST         ← Ch. 7 exemplar — Taro's correct output type
> TOPOLOGY: SINE_WAVE              ← Voss only
```

**Interference results**
```
> INTERFERENCE: CONSTRUCTIVE / FREQUENCY_SHIFT: OCTAVE_DROP   ← Ch. 5 exemplar
> INTERFERENCE: DESTRUCTIVE / ATTACK: NULLIFIED               ← Ch. 7 exemplar
```

**Team link / node status**
```
> TEAM_LINK: STATUS
> NODE_STATUS: NAME [STATUS]       ← Ch. 5: > NODE_STATUS: VEN [CHARGED]
> SYNC: LOCKED [Modifier]          ← replaces > CONNECTION: ESTABLISHED
> DRIFT: VALUE                     ← replaces > LATENCY: VALUE
> LINK: ESTABLISHED [STATUS: STABLE]   ← replaces > HANDSHAKE: COMPLETE
```

**Trap / threat**
```
> TRAP_DETECTED: TYPE [STATUS: CONTAINED]   ← Ch. 7 exemplar
> THREAT_LEVEL: LEVEL
> THREAT_LEVEL: LOW                          ← Ch. 14 exemplar
```

**Physics errors**
```
> WARNING: FEEDBACK_LOOP [INTEGRITY: XX%]   ← Ch. 5 exemplar
> ERROR: PHASE_MISMATCH [TARGET_PHASE: 0] [INCOMING_PHASE: 90]   ← Ch. 5 exemplar

```

### CAT-07: ENVIRONMENT / ZONE

Block-level headers used at the start of scenes. These are distinct from inline HUD tags — they are the scene's "operating parameters."

```
ZONE: LOCATION [SECTION]
ENVIRONMENT: TYPE
TEMP: VALUE [WARNING]
STATUS: CONDITION
PHYSICS: NULL_STATE [SIMULATION]   ← Ch. 5 Sandbox header
OBJECTIVE: DESCRIPTION             ← Ch. 5 Sandbox header
```

**NOTES**: Zone headers are not inline HUD — they are scene-level metadata. They do not substitute for active inline tagging. Chapters 6–8 used these headers without inline HUD content — this is insufficient for Phase II.

### CAT-08: THE SILENCE (Special Category)

The Silence is the absence of signal. Its HUD representation is the absence of data, not a new type of data.

```
> ENEMY: [REDACTED_ANOMALY]        ← Ch. 8 exemplar — confirmed exemplary
> SIGNAL: ABSOLUTE_SILENCE         ← Ch. 8 exemplar — confirmed exemplary
> ANOMALY_DETECTED / SIGNAL_RETURN: NULL    ← Ch. 8 exemplar
> DAMPING: EXTERNAL_SOURCE
> WARNING: SIGNAL_VOID
```

**Glitch format for Silence sequences:**
```
> SI_GNAL L__OSS // N_ULL
> PR_OCES_SING...
> B_UFFER [C_ORRUPT]
> EN_EMY: [__________]
```

**CRITICAL**: The Silence must never be tagged with a presence tag (`> ENTITY:`, `> FORCE:`, `> BEING:`). It is the negation of data. The horror comes from what The Score *cannot* render.

## PART 2: PERMITTED SENSATION: EXCEPTIONS

Only these two `SENSATION:` tags are permitted:

```
> SENSATION: INTEGER_OVERFLOW   ← technically precise — numerical overflow
> SENSATION: NULL               ← technically precise — absence state
```

All other `SENSATION:` tags are prohibited. See replacement table below.

## PART 3: PROHIBITED TAGS — FULL REPLACEMENT TABLE

### Prohibited: Soft/Emotional Register

| Prohibited tag | Reason | Approved replacement |
|---|---|---|
| `> SENSATION: PAIN` | Too soft/subjective | `> STATUS: NERVE_DAMAGE [CRITICAL]` |
| `> SENSATION: NAUSEA` | Too soft | `> BIOMETRIC: VAGAL_RESPONSE [SIGNAL: CORRUPTED]` |
| `> SENSATION: BURNING_WIRE` | Too metaphorical | `> STATUS: THERMAL_LOAD [REGISTER: HIGH // RATE: CRITICAL]` |
| `> SENSATION: BOOT_SEQUENCE` | Too poetic | `> PROCESS: NAI_BOOT [STAGE: 1/3]` |
| `> SENSATION: GOLDEN_SYRUP` | Poetic, not clinical | `> AMBIENT: MIDDLE_C [EFFECT: ENTRAINMENT]` |
| `> SENSATION: FREE_FALL` | Metaphorical | `> STATUS: LIMITER_REMOVED [AMPLITUDE: UNCAPPED]` |
| `> FEELING: ANGRY` | Prohibited prefix | `> MICRO-EXPRESSION: RAGE [SUPPRESSED]` |
| `> THOUGHT: CONFUSED` | Prohibited prefix | `> LOGIC_ERROR // CONTRADICTION_DETECTED` |
| `[PAIN] [PAIN] [PAIN]` | No clinical framing | `> BIOMETRIC: PAIN_INDEX [CRITICAL]` |

### Prohibited: Networking Jargon

| Prohibited | Reason | Approved replacement |
|---|---|---|
| `> CONNECTION: ESTABLISHED` | Networking jargon | `> SYNC: LOCKED [STATUS: STABLE]` |
| `> LATENCY: 0ms` | Networking jargon | `> DRIFT: ZERO [LOCK: CONFIRMED]` |
| `> HANDSHAKE: COMPLETE` | Networking jargon | `> LINK: ESTABLISHED` |
| `> PING: VALUE` | Networking jargon | `> PULSE: VALUE` |
| Any `BANDWIDTH` | Networking jargon | `CAPACITY`, `FLOW`, `BREADTH` |
| Any `PACKET(S)` | Networking jargon | `SIGNAL`, `DATA_STREAM`, `PULSE` |
| Any `SERVER` | Networking jargon | `GRID`, `NODE`, `ANCHOR` |
| Any `DOWNLOAD`/`UPLOAD` | Networking jargon | `RECEIVE`, `TRANSMIT` |

## PART 5: HUD DENSITY GUIDE BY PHASE

| Phase | Chapters | HUD density | Character | What to look for |
|---|---|---|---|---|
| I — Impulse | 1–3 | Dense | Reader onboarding — every event gets a tag | New-person tags, system error codes, boot sequences |
| II — Propagation | 4–8 | Full but efficient | Taro's system is maturing — tags are more precise | Combat data, team link status, social scans |
| III — Interference | 9–11 | Degraded / corrupted | System is failing — glitch format intrudes | Null returns, `[REDACTED]`, corrupted tag names |
| IV — Feedback Loop | 12–14 | Fragmentary / stripped | Post-compliance interface is minimal and cold | Very few tags; no musical aesthetics; clinical only |
| V — Equilibrium | 15–17 | Sparse and earned | Reader is fluent; only critical tags needed | Key moments only; long prose stretches without any HUD |

**Volume One HUD gap**: Chapters 6–7 (partial), 9, 10, 11, 13, 15, 16 show minimal or no HUD in the live manuscript. Chapters 6–11 are Phases II and III — this is a critical deficit. Phase V absence (Ch. 15–16) may be intentional but must be confirmed.
