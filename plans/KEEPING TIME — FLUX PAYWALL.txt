## ⚡ EXECUTIVE SUMMARY

The paywall is not a paywall. It is a **narrative event**.

When the reader reaches the end of Chapter 6 ("Ground State"), the C-Order's Neural-Acoustic
Interface detects their proximity to restricted frequencies and issues a **Damping Order** —
cutting off the Upper Register (Chapters 7–17). The reader's only path forward is to contact
a Flux operative, acquire a **Sync Key** via the black market (Orange Money), and jailbreak
their NAI to unlock what the C-Order suppressed.

Paying P25 via Orange Money is, within the fiction, an **act of resistance.**

The SNS reacts throughout: surveillance spikes when the modal opens, and a gold Alignment
Event fires when access is granted — because the Score always notices.

## 📐 GATE POINT RATIONALE

**Free:** Chapters 1–6 (The Impulse / Ground State)
**Locked:** Chapters 7–17 (Propagation → Equilibrium)

**Why after Ch06?**
- Ch06 is titled "Ground State" — Taro's first experience of consciousness *outside* the Score.
  Thematically, this is where the story pivots from compliance to resistance.
- The audio engine comment in the source confirms: *"Ch06 = the chapter where Taro first
  experiences consciousness outside the Score."*
- Surveillance at Ch06 = 0.2 (lowest in the first act). The reader feels safe. Then the wall
  hits. The tonal whiplash is intentional and diegetically coherent.
- Ch07 surveillance = 0.6, Ch08 = 0.9 — the locked content escalates dramatically, which
  justifies the value of the purchase.

## PHASE 5 — VERIFICATION & UNLOCK (JS)

**`FLUX_CONFIG` object:**
```javascript
const FLUX_CONFIG = {
  // TODO: Replace before going live
  ORANGE_NUMBER:   "7X XXX XXX",        // Your Orange Money number
  PRICE_CREDITS:   25,                   // Amount in Pula (displayed as "Pulse Credits")
  FORMSPREE_ID:    "YOUR_FORM_ID",       // From formspree.io
  UNLOCK_HASH:     "REPLACE_WITH_HASH",  // SHA-256 of your 6-digit Sync Key
  DEBUG_MODE:      true,                 // Set FALSE before going live
  DEBUG_KEY:       "123456"              // Test code (only works when DEBUG_MODE = true)
};
```

**`verifyKey()` logic:**
1. Read entered code from input field
2. If `DEBUG_MODE` is true → compare plain text to `DEBUG_KEY`
3. If `DEBUG_MODE` is false → hash with `crypto.subtle.digest('SHA-256')`, compare to `UNLOCK_HASH`
4. On match → call `unlockUpperRegister()`
5. On mismatch → display `> SYNC KEY REJECTED // CARRIER WAVE MISMATCH`

**`unlockUpperRegister()` logic:**
1. Set `localStorage.setItem('kt_upper_register', 'true')`
2. Remove `#frequency-wall` from DOM
3. Show `#locked-chapters-wrap` (set `display: block`)
4. Close the modal
5. Trigger SNS Alignment Event (see Phase 6)
6. Rebuild NARRATIVE_DATA to include ch07–ch17 beats
7. Scroll reader to top of ch07

## PHASE 7 — LOCALSTORAGE PERSISTENCE

**On `window.onload`**, before `compileNarrative()` is called:
```javascript
// Check if reader has already purchased access
// If yes: skip the wall entirely, compile all 17 chapters as normal
function checkStoredAccess() {
  if (localStorage.getItem('kt_upper_register') === 'true') {
    // Silently unlock — no fanfare on return visits
    document.getElementById('frequency-wall')?.remove();
    const wrap = document.getElementById('locked-chapters-wrap');
    if (wrap) wrap.style.display = 'block';
  }
}
```

This runs before `compileNarrative()`, so returning readers get all 17 chapters compiled
into `NARRATIVE_DATA` without any interruption.
